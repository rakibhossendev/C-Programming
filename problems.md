# 50 Problem Challenge in C (Logic Master)

Goal: Improve logical thinking and problem-solving using C.

---

Level 1: Basic Logic (1–10)

1. Print Hello World without using printf
2. Swap two numbers without using a third variable
3. Check if a number is even or odd
4. Find the largest of three numbers
5. Check if a number is prime
6. Print multiplication table
7. Calculate factorial using loop
8. Reverse a number
9. Check palindrome number
10. Count digits in a number

---

Level 2: Loop Mastery (11–20)

11. Print Fibonacci series
12. Find sum of digits
13. Armstrong number check
14. Strong number check
15. Perfect number check
16. Print star pyramid pattern
17. Print inverted pyramid
18. Print diamond pattern
19. Find GCD of two numbers
20. Find LCM of two numbers

---

Level 3: Array Thinking (21–30)

21. Find maximum in array
22. Find minimum in array
23. Reverse an array
24. Find second largest element
25. Remove duplicate elements
26. Rotate array left
27. Rotate array right
28. Find frequency of elements
29. Merge two arrays
30. Find missing number in array

---

Level 4: String Logic (31–40)

31. Reverse a string
32. Check palindrome string
33. Count vowels and consonants
34. Remove spaces from string
35. Find string length without strlen
36. Convert lowercase to uppercase
37. Check anagram strings
38. Count words in a sentence
39. Remove duplicate characters
40. Find most frequent character

---

Level 5: DSA Starter (41–50)

41. Implement stack using array
42. Implement queue using array
43. Implement linked list
44. Insert node in linked list
45. Delete node from linked list
46. Implement binary search
47. Implement bubble sort
48. Implement selection sort
49. Implement insertion sort
50. Find pair with given sum in array

---

Challenge Rules

1. Solve without copying code.
2. First write algorithm on paper.
3. Then implement in C.
4. Optimize the solution later.
5. Try at least 3 problems daily.

---

Expected Result

After finishing this challenge you will gain:

Strong problem-solving ability
Better algorithm thinking
Confidence in Data Structures
